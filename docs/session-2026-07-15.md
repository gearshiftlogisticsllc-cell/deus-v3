# Session 2026-07-15 — Gmail OAuth & Email Deliverability

## Issues Found

### 1. Gmail API scope mismatch (`gmail.send` → `gmail.modify`)
- Old token had `gmail.send` scope, code requested `gmail.modify`.
- Token refresh failed with `invalid_scope: Bad Request`.
- **Fix:** Updated scope in `gmail_sender.py:54` and `app/api/routes.py:1148,1186`.

### 2. GmailSender singleton cached `available=False` permanently
- `GmailSender` initialized at app startup before any OAuth token existed.
- `_init_service()` set `self._available = False` and `get_gmail_sender()` returned the stale singleton forever.
- Even after successful OAuth callback saved a new token to DB, the singleton never rechecked.
- **Fix:** `gmail_sender.py:211` — `get_gmail_sender()` now creates a fresh instance if `available` is `False`.

### 3. Invalid Gemini API key
- `GEMINI_API_KEY` env var contained an expired/invalid key.
- All Gemini calls returned `API key not valid`.
- **Fix:** Regenerate key at https://aistudio.google.com/app/apikey and update Railway env var.

### 4. Invalid Groq API key (now removed)
- `GROQ_API_KEY` env var contained an invalid key.
- All Groq calls returned `Invalid API Key`.
- **Fix:** Remove the `GROQ_API_KEY` variable entirely from Railway env vars.

### 5. SMTP blocked on Railway
- Outbound SMTP ports (465, 587) are blocked by Railway's network policy.
- `[Errno 101] Network is unreachable` on both ports.
- **Resolution:** Rely on Gmail API (HTTP-based, not blocked) as primary send method.

### 6. No Resend API key configured
- Resend was the last fallback in `email_sender.py` but no key was set.
- **Not an issue** — Gmail API should be sufficient once token is valid.

### 7. Spam trigger words in default subject
- `outreach_config.py` default subject: "Quick proposal for {business_name}"
- SpamChecker weights: "proposal" (+3), "quick call" (+2)
- **Fix:** Changed to "Quick opportunity for {business_name}"

### 8. 6 of 7 leads skipped in prior batch
- Caused by Gmail daily cap redirect (misleading code), rate limiter (hourly cap 10/hr), and no batch delay between sends.
- **Fix:** Added batch delay in `outreach_agent.py`, removed misleading Gmail cap redirect to scout mode.
