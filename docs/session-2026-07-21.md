# Session 2026-07-21 ŌĆö Login Fix, Lead Scout Overhaul, Brain CLI

## Issues & Resolutions

### 1. Login returning "Connection error" (500 with empty body)
- `auth_routes.py:104` declared `request: LoginRequest` ŌĆö this bound `request` to the Pydantic body model, **not** the Starlette `Request` object.
- `request.client.host` on line 105 raised `AttributeError` before any try/except, causing FastAPI to return 500 with empty body.
- Frontend `.catch()` handler in `login.html` hit `r.json()` on the empty response, throwing "Connection error".
- **Fix:** Renamed body param to `login_data`, added separate `http_request: Request = None` for the HTTP object. Updated all internal references.

### 2. Lead scout finding 0 leads on Railway
- DuckDuckGo + Google direct scraping (free sources) are blocked by cloud IPs (Railway, AWS, etc.) ŌĆö they return empty results.
- The daemon and API routes created `LeadScoutAgent(None, None, llm)` ŌĆö passing `None` for both BrightData and Serper even when `SERPER_API_KEY` was set in env.
- The `/api/lead-scout/search` endpoint required `use_serper: true` from the frontend to use Serper, defaulting to `false`.
- **Fix:** All code paths now use `SerperSource(SERPER_API_KEY) if SERPER_API_KEY else None` ŌĆö automatically uses Serper when key exists. Removed `use_serper` frontend flag gating.

### 3. Daemon rotation stuck on "pending"
- `_run_lead_scout_rotation()` read `os.getenv("LEAD_SCOUT_NICHE")` ŌĆö this env var was never set on Railway.
- The rotation only ran between 12:00-12:09 PM ŌĆö if daemon started any other time, it never triggered.
- The daemon_config DB table had the niche (`"Hvac companies hiring administrative roles"`) but the rotation code never read it.
- **Fix:** Rotation now falls back to daemon_config DB ŌåÆ PDF rules. Removed the 12 PM time gate ŌĆö runs every cycle, gated by `is_state_completed_today()` which enforces one state per day.

### 4. Uploaded rules PDF niches not followed
- `_load_niches_from_rules()` returned raw lines from PDF text as search queries.
- PDF contained ICP descriptions ("Target Verticals: HVAC Companies...", "Company Size: 1-10 employees") plus unrelated PDF boilerplate text.
- The agent searched for lines like "Download Adobe Acrobat Reader" and got irrelevant results.
- **Fix:** `_load_niches_from_rules()` now uses the LLM to extract 1-3 actual search queries from the rules text. Rules context is also injected into the scoring prompt so leads are scored against ICP criteria.

### 5. Rules PDF: only first line used
- Original fallback only took `lines[0]` from the rules PDF, ignoring additional niches.
- **Fix:** When multiple niches are extracted from rules, each is searched separately and results are combined + deduplicated.

### 6. Lead scout results not paginated
- `renderScoutTable()` dumped all leads (200+) into one flat table with no pagination or batch download.
- **Fix:** Replaced with paginated table (25/50/100 per page via "Batch Page" dropdown). Added Prev/Next navigation. Added "Download Page CSV" and "Download All CSV" buttons.

### 7. No Brain CLI for chatting with LLM
- `brain.py` was just a test harness with no REPL, no conversation memory, no agent execution.
- `nexus_bridge.py` had a basic CLI but no Gemini failover, no chat history, no rules injection.
- **Fix:** Added full REPL to `brain.py` (`python brain.py`) ŌĆö conversational chat with history, GroqŌåÆGemini failover, intent detection for agent commands, `/help` `/agents` `/clear` `/status` commands. Injects rules PDF context into every turn.

### 8. Serper not used in geo auto-scout and daemon paths
- `auth_routes.py:663`, `routes.py:1424`, `daemon.py` Source 2+3 all created agents without Serper.
- **Fix:** Serper added to all geo-target and daemon scout code paths.
